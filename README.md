# Finance Flow Bot

Telegram-бот для учета личных финансов: доходов, расходов, счетов, категорий и бюджетов.

Бот помогает:
- контролировать движение денег;
- создавать виртуальные счета;
- переводить деньги между счетами;
- вести категории трат;
- ставить бюджеты по категориям;
- отслеживать превышения бюджета;
- смотреть историю операций.

---

## Возможности

### Финансы
- создание банковских счетов;
- просмотр списка счетов и балансов;
- установка счета по умолчанию;
- переименование счета;
- удаление счета;
- перевод денег между счетами;
- добавление операций дохода и расхода;
- отмена последней операции;
- добавление описания к операции.

### Категории
- создание категорий;
- удаление категорий;
- использование категорий при добавлении операций;
- автосвязывание ключевых слов с категориями.

### Бюджеты
- создание бюджета для категории;
- редактирование бюджета;
- удаление бюджета;
- просмотр всех бюджетов;
- отображение прогресса в формате `потрачено / лимит`;
- предупреждения при достижении 80%, 90% и 100% бюджета.

### История
- просмотр истории операций;
- календарь для выбора даты;
- просмотр операций за конкретный день.

### Уведомления
- предупреждение о перерасходе бюджета;
- предупреждение, если баланс счета уходит ниже нуля.

## Сценарии работы

### 1. Регистрация
Пользователь нажимает кнопку **«Зарегистрироваться»**.  
После этого бот создает пользователя и предлагает создать первый виртуальный банковский счет.

### 2. Создание счета
Пользователь выбирает **«Создать счет»**, вводит название, и счет появляется в списке счетов.

### 3. Добавление операции
Пользователь вводит сумму:
- положительное число — доход;
- отрицательное число — расход.

Примеры:
- `300 еда`
- `-1500 такси`

Если категория указана, бот пытается найти ее по ключевому слову.  
Если категория не найдена, бот предлагает выбрать ее вручную.  
После добавления операции можно:
- добавить описание;
- отменить операцию.

### 4. Перевод между счетами
Пользователь выбирает два счета:
- первый — счет списания;
- второй — счет пополнения.

После этого бот создает две операции:
- расход на одном счете;
- доход на другом.

### 5. Бюджеты
Пользователь может:
- создать бюджет для категории;
- изменить бюджет;
- удалить бюджет;
- посмотреть прогресс расходов по всем категориям.

### 6. История
Пользователь открывает календарь, выбирает дату и смотрит операции за день.

---

## Текущая логика бюджета

Бот отслеживает расход по бюджету и может показать один из статусов:
- зеленый — все нормально;
- оранжевый — потрачено более 80%;
- красный — перерасход.

Также отображается общий прогресс по всем бюджетам пользователя.

---

## Автокатегоризация

В проекте уже заложена база для ключевых слов категорий.

Пример логики:
- если пользователь пишет `uber`, операция может быть отнесена к транспорту;
- если `steam`, операция может быть отнесена к играм.

Если совпадение не найдено, бот предлагает выбрать категорию вручную и может сохранить новую связь ключевого слова с категорией.

## Стек технологий

- **Python**
- **aiogram**
- **FSMContext** для диалогов и шагов сценария
- **SQLAlchemy / ORM** или другая DB-логика проекта
- **InlineKeyboardBuilder / ReplyKeyboardBuilder** для меню
- **dotenv** для переменных окружения

---

## Структура логики проекта

По коду проект разделен на несколько слоев:

- **handlers** — обработка сообщений, callback'ов и сценариев пользователя;
- **keyboard** — кнопки и клавиатуры;
- **services** — бизнес-логика;
- **crud** — работа с базой данных;
- **domain / enums** — перечисления и доменные сущности;
- **static** — состояния FSM.

## Особенности реализации

- добавление операций без обязательного выбора счета в текущем сценарии;
- проверка регистрации перед основными действиями;
- защита от некорректного ввода суммы;
- поддержка состояний для поэтапного ввода данных;
- логирование важных действий через `logging`;
- автоматическая проверка превышения бюджета при добавлении операции.

## Запуск

Ниже пример шаблона запуска. Подставь сюда реальные команды своего проекта:

```bash
git clone <repo-url>
cd <project-folder>
python -m venv venv
source venv/bin/activate  # Linux / macOS
# или
venv\Scripts\activate     # Windows

pip install -r requirements.txt
```

Создай .env файл и укажи необходимые переменные окружения:

```
BOT_TOKEN=your_telegram_bot_token
DATABASE_URL=your_database_url
CATEGORY_FOR_TRANSFER=transfer_category_name
```

Запуск:
```
python main.py
```
